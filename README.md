## 📌 Featured Projects

### 1. Highly Available 3-Tier Web Architecture (Infrastructure as Code)
*Terraform | AWS (VPC, EC2, ALB, RDS) | IAM Security*

Designed and provisioned a production-ready, fault-tolerant 3-tier architecture entirely through code to ensure reproducibility and disaster recovery.

**Production-Level Features:**
- Automated the deployment of a custom VPC with isolated public and private subnets across multiple Availability Zones.
- Implemented an Application Load Balancer (ALB) and Auto Scaling Groups (ASG) for high availability and traffic distribution.
- Secured backend databases (RDS) in private subnets, accessible only via bastion hosts and strict security group rules.
- Managed Terraform state securely using remote Amazon S3 backends with DynamoDB state locking.

### 2. End-to-End Containerized CI/CD Pipeline
*GitHub Actions | Docker | AWS ECR & ECS*

Built an automated, zero-downtime deployment pipeline for a microservices application, emphasizing security and continuous integration.

**Production-Level Features:**
- Configured GitHub Actions workflows to trigger automated tests and builds on every pull request.
- Automated the pushing of tagged Docker images to Amazon Elastic Container Registry (ECR).
- Handled rolling updates to Amazon Elastic Container Service (ECS) to ensure zero-downtime deployments.

### 3. AWS IAM Security & Access Auditor (Python)
*Python | AWS Boto3 | IAM | CloudTrail | DevSecOps*

Developed `elasticache_auditor.py`, a custom Python security automation tool designed to enforce the principle of least privilege (PoLP) across AWS environments by auditing access to ElastiCache clusters.

**Production-Level Features:**
- Automated the scanning of all managed and inline IAM policies to map theoretical access permissions to specific users and roles.
- Integrated with **AWS CloudTrail** to query the last 7 days of API events, effectively distinguishing between users who just have permissions versus those actively utilizing them.
- Generated structured JSON audit reports to provide actionable security insights, identifying over-provisioned access rights and improving overall cloud security posture.

### 4. AI-Powered Serverless CI/CD Pipeline
*Final Year Project – University of Management and Technology*

Built a **serverless CI/CD system using AWS services** designed to optimize development workflows.

**Key Features:**
- Automated build, testing, and deployment pipelines.
- AI-powered error detection before code commits.
- Cloud-native architecture using AWS Fargate, Amazon ECS, API Gateway, DynamoDB, and Amazon S3.
